# PLAN-002 Dashboard 易用性优化

## 背景

当前 Dashboard 已具备队列查看、生命周期操作、诊断和投递能力，但截图中的默认体验仍有几个低认知负担问题：

- 默认选中历史队列 `no_namespace`，主区域显示“没有任务”，但左侧实际存在有积压的队列。
- 队列列表按名称展示，异常和积压队列不够突出。
- 无可操作任务时，重试、重放、恢复、清空等按钮仍可点击。
- “全部”容易被理解为包含历史，当前页面更适合表达“当前任务”。
- 投递任务表单占据右侧主要空间，但多数运维场景首先是观察和处理积压。

## 目标

- 打开页面后优先看到最需要处理的队列。
- 让“当前任务”和“历史任务”在界面语义上分开。
- 禁用当前无效的危险或批量操作。
- 降低左侧队列扫描成本，优先展示有当前任务或 Worker 状态的队列。
- 保持现有后端 API 和数据模型不变。

## 实施方案

1. 前端增加队列优先级排序，按 DLQ、retry、stale worker、processing、ready、delay、history 的运维价值排序。
2. 初始队列选择改为选择最高优先级队列，而不是列表第一项。
3. 左侧增加“只看当前”开关，默认优先展示有当前任务或 Worker 的队列。
4. 状态标签中文化，并把 `all` 展示为“当前”。
5. 状态 tab 显示各状态数量，帮助用户不用切换也能判断有没有内容。
6. 队列操作按钮根据统计值禁用无效动作。
7. 空状态根据当前过滤条件给出更准确的提示。
8. 投递任务区域改为折叠面板，避免抢占诊断和任务列表注意力。

## 验证

- 运行 JS 模块语法检查。
- 运行 `ruff`、`mypy`、相关 pytest。
- 如环境允许，用浏览器打开 Dashboard 做视觉和交互确认。
