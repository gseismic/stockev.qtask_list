# PLAN-003 Dashboard 问题处理控制台

## 目标

把 Dashboard 从“状态展示面板”推进到“问题处理控制台”，降低真实运维场景下的认知负担。

## 实施内容

1. 增加全局连接错误横幅，连接失败时标记数据可能过期。
2. 将写操作统一受连接状态控制，连接异常时禁用批量操作、危险操作、投递和单任务写操作。
3. 在主区域增加当前队列问题横幅，突出：
   - 待处理积压但没有活跃 Worker
   - DLQ 失败任务
   - retry 待重试任务
   - stale Worker processing
4. 明确顶部统计口径为“当前队列状态”。
5. 把任务表改成“任务样本”，在大量积压且无 Worker 的场景下默认折叠。
6. 把 `强制恢复` 和 `清空` 移入危险操作折叠区。
7. 保持现有 API 和后端数据模型不变。

## 验证

- JS 模块语法检查通过。
- `ruff`、`mypy`、`pytest` 通过。
- 浏览器验证：
  - 桌面宽度无 console error。
  - 积压无 Worker 时显示问题横幅。
  - 危险操作默认折叠。
  - 移动宽度无页面级横向溢出。
